# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest | :x:               |

We only provide security updates for the latest version of Subterfuge. Please ensure you are running the most recent version available on the App Store or Google Play.

## Reporting a Vulnerability

We take the security of Subterfuge seriously. If you discover a security vulnerability, please follow these steps for responsible disclosure:

### How to Report

**Do NOT report security vulnerabilities through public GitHub issues.**

Instead, please email us at: **security@secretplan.com**

### What to Include

Please include the following information in your report:

- **Description**: A clear description of the vulnerability
- **Impact**: The potential impact and severity of the vulnerability
- **Steps to Reproduce**: Detailed steps to reproduce the issue
- **Affected Versions**: Which version(s) of the game are affected
- **Platform**: iOS, Android, or Web
- **Proof of Concept**: Any code, screenshots, or videos that demonstrate the vulnerability
- **Suggested Fix**: If you have ideas on how to fix the issue (optional)

### Response Timeline

- **Acknowledgment**: We will acknowledge receipt of your report within **48 hours**
- **Initial Assessment**: We will provide an initial assessment within **7 days**
- **Resolution**: We aim to resolve confirmed vulnerabilities within **14 days**, depending on severity and complexity

### What to Expect

1. We will investigate your report and determine its validity
2. If confirmed, we will work on a fix and keep you updated on progress
3. Once resolved, we will credit you in our release notes (unless you prefer to remain anonymous)
4. We may ask for additional information or clarification during the process

## Scope

### In Scope

- Authentication and authorization vulnerabilities
- Data exposure or leakage
- Account takeover vulnerabilities
- Game manipulation that affects other players
- Payment/IAP security issues
- Server-side vulnerabilities
- Client-side vulnerabilities with real security impact

### Out of Scope

- Social engineering attacks on staff or users
- Physical attacks
- Denial of service (DoS/DDoS) attacks
- Spam or abuse issues
- Game balance issues (report these on [Discord](https://discord.gg/TGZ2kFTDQV))
- Bugs that do not have a security impact
- Issues in third-party services or applications

## Safe Harbor

We will not pursue legal action against researchers who:

- Make a good faith effort to avoid privacy violations and disruptions to others
- Do not exploit the vulnerability beyond what is necessary to demonstrate it
- Report the vulnerability to us before disclosing it publicly
- Give us reasonable time to address the issue before any public disclosure

Thank you for helping keep Subterfuge and our players safe!
